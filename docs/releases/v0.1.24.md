# v0.1.24

## 优化

- 为 OpenAI Images 兼容的图生图接口增加受控的 multipart 兼容协商，可在 `image`、`image[]` 和精简单图表单之间安全适配。
- 兼容方式验证成功后会按供应商、接口、模型和请求配置记忆，后续请求直接使用已验证方式。

## 修复

- 修复部分 New API/第三方网关在 multipart 图生图请求中偶发报告 `model 为空` 时无法自动恢复的问题。
- 只有明确的 HTTP 400 请求参数校验失败才会进行有界兼容回退；断网、超时、408、429、5xx 和未知错误不会自动重复提交，避免重复生图和重复计费。
- 修复响应侧 `content-type`、HTTP 200 业务失败等情况可能被误识别为可重试或被错误记忆的问题。
